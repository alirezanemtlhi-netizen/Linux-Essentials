# Linux Essentials — Day 4
## Chapter 4: Desktops, Apps & Languages

### 1. Desktop Environments (DE)
* **KDE:** Best for Windows/Mac users (looks familiar).
* **GNOME:** Best for absolute beginners (simple and clean).
* **XFCE:** Best for old PCs and advanced users (light and fast).
* **Note:** You can install multiple DEs at the same time.

### 2. Apps & Windows Equivalents
* **Browser:** Firefox / Chrome / Brave
* **Email:** Thunderbird (Outlook alternative)
* **Office:** LibreOffice (MS Office alternative)
* **Graphics:** Blender (3D graphics) & VLC (Video player)

### 3. Programming Languages
* **Assembly:** Low-level, hardest, non-portable (hardware-specific).
* **C:** The main language of the Linux Kernel. Uses **GCC**. Fast but hard to debug.
* **C++:** Updated version of C (object-oriented).
* **Java:** Cross-platform, uses both compiled and interpreted methods.
* **Python:** Very popular, interpreted, easy to read. Great for scripting.
* **Perl:** Interpreted language for text processing.
* **PHP:** Used for web development (works with Apache).

#### Compiled vs Interpreted
* **Compiled (C/C++):** Converts all code at once. Very fast.
* **Interpreted (Python):** Translates code line-by-line. Slower but easier to debug.
* **Shell Scripting:** Not a full language. Used to automate tasks in the terminal.

### 4. Package Managers & Repositories
* **Repository (Repo):** Central servers that hold software (e.g., Kali Repo is 2TB+).
* **Dependencies:** Shared files that software needs to run.
* **Debian/Ubuntu:** Uses `.deb` packages with `dpkg` and `apt` / `apt-get`.
* **Red Hat/Fedora:** Uses `.rpm` packages with `rpm` and `yum` / `dnf`.
